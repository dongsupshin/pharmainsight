APPLICATION WORKFLOW
================================================================================

1. USER IMPORTS DOCUMENT
   → MainViewModel.ImportDocumentCommand
   → File dialog selection
   → PromoDocument instance created
   → Added to ObservableCollection

2. USER ANALYZES DOCUMENT
   → MainViewModel.AnalyzeDocumentCommand
   → DocumentParserService.ParseDocumentAsync()
   → ClaimsDetectorService.DetectClaims()
   → ComplianceCheckerService.CheckCompliance()
   → Score calculation
   → Document status updated

3. USER REVIEWS RESULTS
   → DocumentReviewView displayed
   → Claims shown in list
   → Compliance issues shown in list
   → User marks claims as approved/flagged
   → User resolves issues
   → Review notes added

4. USER EXPORTS REPORT
   → ReportGeneratorService.GenerateTextReport()
   → or ReportGeneratorService.GenerateExcelReport()
   → File saved to user location
   → Report ready for MLR submission

5. USER VIEWS DASHBOARD
   → DashboardViewModel.UpdateMetrics()
   → Summary statistics displayed
   → Recent activity shown
   → Score distribution visualized

COMPLIANCE RULES IMPLEMENTED
================================================================================

FAIR BALANCE CHECK:
  - Risks section must exist
  - Risks should be >= 1/3 of benefits size
  - Flag if missing or disproportionate

ISI VALIDATION:
  - Check for Important Safety Information
  - Verify contraindications present
  - Verify warnings present
  - Verify adverse reactions present

BLACK BOX WARNING:
  - Identify drug categories requiring boxed warning
  - Verify warning is included
  - Flag if missing for restricted drugs

REFERENCE SUBSTANTIATION:
  - All medium/high/critical claims need references
  - Check for citation indicators
  - Flag unsubstantiated claims per type

OFF-LABEL DETECTION:
  - Flag expanded use language
  - Check for unapproved indication references
  - Identify comparative claims without context

ADVERSE EVENTS:
  - Require dedicated adverse events section
  - Verify event list completeness
  - Check serious event inclusion
  - Validate frequency information

MISLEADING PRESENTATION:
  - Flag unqualified superlatives
  - Check qualifier adequacy
  - Identify cherry-picked data patterns
  - Assess risk minimization language

================================================================================
CLAIM DETECTION PATTERNS
================================================================================

EFFICACY PATTERNS:
  - Effectiveness: proven, demonstrated, significant improvement
  - Performance: reduces, decreases, improves, enhances
  - Statistics: statistically significant, p<0.05, CI, confidence interval
  - Comparative: works better, more effective

SAFETY PATTERNS:
  - Tolerability: well-tolerated, minimal side effects, low risk
  - Absence: no serious, lacks, without adverse, free from
  - Safety: safe, gentle, safe in, tolerable

SUPERIORITY PATTERNS:
  - Category: first-in-class, best-in-class, gold standard
  - Uniqueness: only drug, breakthrough, revolutionary
  - Excellence: superior, unmatched, unprecedented

COMPARATIVE PATTERNS:
  - Direct: better than, superior to, more effective than
  - Competitive: unlike other, advantage over
  - Head-to-head: versus, compared to

ECONOMIC PATTERNS:
  - Cost-effectiveness: cost-effective, saves money
  - Financial: reduces healthcare costs, affordable
  - Value: return on investment, economical

COMPLIANCE SCORE RANGES
================================================================================

90-100: EXCELLENT
  Status: Approved
  Assessment: Ready for MLR submission
  Action: Submit as-is

75-89: GOOD
  Status: Reviewed
  Assessment: Minor revisions recommended
  Action: Address identified issues

60-74: FAIR
  Status: Reviewed
  Assessment: Moderate revisions required
  Action: Make substantial edits

Below 60: POOR
  Status: Flagged
  Assessment: Major revisions needed
  Action: Comprehensive review required

================================================================================
ISSUE SEVERITY LEVELS
================================================================================

CRITICAL (Red):
  - Blocking issues preventing MLR submission
  - Examples: Missing ISI, missing Black Box warning, off-label claims
  - Action: Must be resolved before submission

MAJOR (Orange):
  - Significant issues requiring revision
  - Examples: Imbalanced benefits/risks, unsubstantiated high-risk claims
  - Action: Should be resolved before submission

WARNING (Yellow):
  - Minor issues that should be addressed
  - Examples: Incomplete adverse event listing
  - Action: Recommended to resolve

INFO (Blue):
  - Informational items
  - Examples: General compliance notes
  - Action: Optional improvements

================================================================================
CLAIM RISK LEVELS
================================================================================

LOW: General statements, well-supported, no comparative language
MEDIUM: Safety claims, economic claims, or modest assertions
HIGH: Efficacy claims, superiority claims, comparative claims
CRITICAL: Unqualified superiority claims, unsupported efficacy claims
